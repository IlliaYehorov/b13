Інструкція по користуванню офісним сервером B13-QNAP-1688

Ви можете користуватись робочими файлами з офісного сервера на своєму домашньому Mac чи PC. Для цього ми використовуємо захищений VPN-вузол. Підключення дозволить вам переглядати, викачувати чи завантажувати свої матеріали напряму на сервер. Монтувати напряму з файлів на сервері не вийде через обмежену швидкість інтернету.

Важливо:
  • Файли, які ви бачите у папках сервера — це єдині оригінали. Якщо ви видалите файл у себе у Finder чи Провіднику, він назавжди видалиться з сервера для всіх.
  • Розміщуйте свої проекти не в кореневій папці диску, а в папках відповідного року.
  • Називайте папки своїх проектів та матеріалів у форматі РРРР-ММ-ДД_Назва-проекту_Автор, наприклад 1979-05-19_Apocalypse-Now_F-Coppola.
  • Коли VPN-тунель увімкнено, весь ваш інтернет-трафік проходитиме через офіс, швидкість вашого інтернету суттєво впаде. Після роботи одразу вимикайте тунель WireGuard.
Крок 1Встановлення VPN клієнта WireGuard

Завантажте та встановіть офіційну програму WireGuard. Вона доступна для MacOS, Windows, iOS та Android.

Крок 3Встановлення конфігурації WireGuard
  • Відкрийте програму WireGuard.
  • Натисніть Import tunnel(s) from file (або Cmd + O).
  • Виберіть отриманий .conf файл та встановіть його.
Крок 4Підключення VPN-тунелю та робота з сервером

Увімкніть VPN (натисніть Activate у WireGuard).

Для macOS (Finder):

Виберіть: Перейти ➔ Під'єднатися до сервера... (або Cmd + K).

smb://192.168.88.200

Вставте адресу та виберіть папку home.

Для Windows (Провідник):

Відкрийте Цей ПК ➔ Підключити мережевий диск.

\\192.168.88.200\home

Вставте цей шлях у поле Папка.

Панель керування сервером через браузер: http://192.168.88.200:8080

Для авторизації встановіть QNAP Authenticator: iOS або Android.

Крок 5Завершення роботи

Після роботи одразу вимкніть тунель WireGuard (натисніть Deactivate), щоб не навантажувати офісну мережу та повернути повну швидкість вашого інтернету.

FAQПроблеми та їх вирішення

Якщо з'єднання не відбувається, то, можливо:

  • В офісі немає світла, та сервер вимкнуто – спробуйте підключитись пізніше, або запитайте когось з офісу чи ввімкнено сервер.
  • У вашого чи офісного (Utels) провайдера проблеми з мережею – спробуйте підключитись пізніше, або дізнайтеся статус мережі в службі підтримки провайдера.
  • Змінилася ваша IP-адреса (динамічний IP) або ви заходите з іншого місця перебування. Ще раз перевірте свою IP адресу на цьому сайті та впевніться, що вона збігається з тою ж, яку ви подавали у білий список IP – в разі зміни постійного місця перебування, подайте свою нову актуальну IP адресу.
